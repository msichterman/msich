## Project Overview

Personal portfolio website for Matt Sichterman at msich.dev. Built on the T3 Stack with Next.js 15 App Router, React 19, TypeScript, tRPC, Prisma, and Tailwind CSS 4.

## Commands

- `npm run dev` — Start dev server with Turbopack
- `npm run build` — Production build
- `npm run lint` — ESLint
- `npx prisma generate` — Regenerate Prisma client (runs automatically on `npm install`)
- No test runner is configured

## Architecture

### App Router Structure

Routes are organized under the `(home)` route group which provides the shared Header/Footer layout:

- `/` — Home page (hero, photos, newsletter, resume)
- `/blog`, `/blog/[slug]` — Blog posts (Fumadocs MDX from `content/blog/`)
- `/notes/[[...slug]]` — Documentation/notes (Fumadocs MDX from `content/notes/`)
- `/contact` — Contact form (react-hook-form + Zod, sends via nodemailer)
- `/about`, `/projects`, `/uses` — Static content pages

API routes:
- `/api/trpc/[trpc]` — tRPC handler (GET/POST)
- `/api/og` — Vercel OG image generation (edge runtime)
- `/api/search` — Fumadocs search endpoint

### tRPC + Prisma Backend

tRPC is configured with SuperJSON transformer and React Query integration. The client setup is in `src/utils/trpc.ts` and provider wiring in `src/app/providers.tsx`.

Routers in `src/server/trpc/router/`:
- **message** — Contact form: sends email via Gmail SMTP (nodemailer)
- **newsletter** — MailerLite API integration for email subscriptions
- **metric** — Personal goal/metrics tracking with Prisma

Prisma uses MySQL with `relationMode = "prisma"` (PlanetScale-compatible). Singleton client pattern in `src/server/db/client.ts`.

### Content (Fumadocs MDX)

Blog posts and notes are MDX files processed by fumadocs-mdx. Collections are defined in `source.config.ts` at the repo root. Content loaders are in `src/lib/source.ts`. Blog posts extend the frontmatter schema with `author`, `date`, and `tags` fields.

**Important:** `source.config.ts` uses `zod4` (aliased as `npm:zod@^4.3.6`) while the rest of the app uses `zod` v3. This is a fumadocs requirement.

### Styling

Tailwind CSS 4 configured via PostCSS (`@tailwindcss/postcss`). Global styles and theme customization live in `src/styles/globals.css` — this includes Fumadocs theme overrides, prose styling for light/dark modes, and custom CSS variables.

Custom fonts: Cal Sans (headings/UI) and IBM Plex Mono (code), loaded in `src/app/layout.tsx`.

Dark mode uses a `.dark` class toggle with a client-side script in the root layout to prevent flash.

## Path Aliases

- `@/*` → `src/*`
- `fumadocs-mdx:collections/*` → `.source/*` (auto-generated by fumadocs)

## Environment Variables

Validated via Zod schemas in `src/env/schema.mjs`:

| Variable | Required | Description |
|---|---|---|
| `DATABASE_URL` | Yes | MySQL connection string |
| `NODE_ENV` | Yes | development / test / production |
| `MAILERLITE_API_KEY` | Yes | Newsletter integration |
| `GMAIL_APP_PASSWORD` | No | Contact form email sending |
| `NEXT_PUBLIC_SITE_URL` | Yes | Public site URL |

Set `SKIP_ENV_VALIDATION=true` to bypass validation during builds.

## Prisma Schema

Three models in `prisma/schema.prisma`: **Contact** (form submissions), **Metric** (goal tracking), **Entry** (metric data points with cascade delete from Metric).
